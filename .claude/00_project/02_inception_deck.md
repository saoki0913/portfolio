# インセプションデッキ

## プロジェクト概要
**ポートフォリオサイト - Modern Tech & Minimalist Precision**

Next.js 15 (App Router) + FastAPI のフルスタックポートフォリオサイト。
単なる自己紹介サイトではなく、モダンなWebエンジニアリング技術の実践場として、高い技術力とUXを両立させる。

## 4. やらないことリスト

プロジェクトの範囲を明確にするため、**やらないこと**を定義する。

### Phase 1では実装しない（将来的に検討）
- ✗ ブログ機能（MDX、Notion API連携） → Phase 2
- ✗ ダークモード/ライトモードトグル → Phase 2（現在はDark Mode固定）
- ✗ 多言語対応（i18n: 日本語・英語） → Phase 2
- ✗ アナリティクス統合（Vercel Analytics, Google Analytics） → Phase 2
- ✗ OGP画像動的生成 → Phase 2

### 永続的にやらないこと
- ✗ ユーザー登録・認証機能（ポートフォリオは公開情報のみ）
- ✗ コメント機能（スパム対策の工数に見合わない）
- ✗ eコマース機能（目的外）
- ✗ SNS連携ログイン（認証不要のため）

### 技術的にやらないこと
- ✗ Pages Routerの使用（App Router固定）
- ✗ CSS Modules、Styled Components（Tailwind CSS固定）
- ✗ Redux、Zustand（Server Componentsで状態管理を最小化）
- ✗ GraphQL（REST API固定、シンプルさ優先）

## 6. 技術的な解決策の概要

### アーキテクチャ
```
┌─────────────────────────────────────────────────┐
│         Frontend (Next.js 15 App Router)        │
│  ┌───────────────────────────────────────────┐  │
│  │  Server Components (SSR)                  │  │
│  │  - Hero, About, Skills, Works sections    │  │
│  │  - SEO Metadata, OGP                      │  │
│  └───────────────────────────────────────────┘  │
│  ┌───────────────────────────────────────────┐  │
│  │  Client Components (Interactive)          │  │
│  │  - Contact Form, Animations              │  │
│  │  - Framer Motion, React Hook Form        │  │
│  └───────────────────────────────────────────┘  │
└─────────────────────────────────────────────────┘
                        ↓ Axios (HTTP Client)
┌─────────────────────────────────────────────────┐
│           Backend (FastAPI)                     │
│  ┌───────────────────────────────────────────┐  │
│  │  API Layer                                │  │
│  │  - RESTful Endpoints                      │  │
│  │  - Pydantic Validation                    │  │
│  └───────────────────────────────────────────┘  │
└─────────────────────────────────────────────────┘
                        ↓ Supabase Client
┌─────────────────────────────────────────────────┐
│           Database (Supabase PostgreSQL)        │
│  - works, skills, about, educations             │
│  - experiences, hero_introduction               │
│  - timeline_items, contact_messages             │
└─────────────────────────────────────────────────┘
```

### データフロー
1. **データ取得**: Server Components → FastAPI → Supabase
2. **フォーム送信**: Client Components → FastAPI → SMTP + Supabase
3. **型の一致**: TypeScript (Frontend) ↔ Pydantic (Backend)

### 技術選定の理由
| 技術 | 選定理由 |
|------|----------|
| Next.js 15 App Router | Server Components によるSEO最適化、高速レンダリング |
| FastAPI | Pythonの型安全性、自動APIドキュメント生成、高速なパフォーマンス |
| Supabase | PostgreSQL、認証、ストレージを統合したBaaS、開発速度の向上 |
| Tailwind CSS 4 | ユーティリティファースト、デザインシステムとの親和性 |
| Framer Motion | 宣言的API、パフォーマンスの良いアニメーション |
| Radix UI | アクセシビリティ対応、カスタマイズ性の高さ |

## 7. 夜も眠れなくなるような問題

### 技術的リスク
1. **パフォーマンス劣化**
   - **問題**: Framer Motionの過度な使用によるレンダリング遅延
   - **対策**: useReducedMotionフック、アニメーション最適化、Lighthouse継続監視

2. **型の不一致**
   - **問題**: Frontend TypeScript型 と Backend Pydantic型のずれ
   - **対策**: 07_type_definitions.mdでの一元管理、定期的な同期確認

3. **Supabase接続エラー**
   - **問題**: DB接続失敗時のアプリケーション停止
   - **対策**: Mock Modeフォールバック、エラーハンドリング設計

### ビジネスリスク
1. **採用担当者のUX理解不足**
   - **問題**: 過度にインタラクティブすぎて「使いにくい」と感じる可能性
   - **対策**: 基本情報は常に表示、アニメーションはオプション（prefers-reduced-motion対応）

2. **SEO対策不足**
   - **問題**: 検索エンジンからの流入が得られない
   - **対策**: Server Componentsによる静的HTML生成、メタデータ最適化

### 運用リスク
1. **メンテナンス工数**
   - **問題**: 個人開発のため、継続的なメンテナンスが困難
   - **対策**: 自動テスト、CI/CD、Mock Mode（DB不要での開発可能）

## 9. 何を諦めるのか

トレードオフの明確化

### 品質 vs 速度
- **選択**: 品質優先
- **理由**: ポートフォリオは「技術力の証明」が目的。速度よりも型安全性・アクセシビリティ・パフォーマンスを優先

### 機能の豊富さ vs シンプルさ
- **選択**: シンプルさ優先
- **理由**: 「何でもできる」より「何が得意か」を明確に伝える。Phase 1では核となる機能のみ実装

### カスタマイズ性 vs 一貫性
- **選択**: 一貫性優先
- **理由**: デザインシステムに従い、統一感のあるUIを実現。過度なカスタマイズは避ける

### 最新技術 vs 安定性
- **選択**: 最新技術（ただし安定版）
- **理由**: Next.js 15、React 19、Tailwind CSS 4は最新だが、ポートフォリオの目的は「最新技術のキャッチアップ力」を示すこと
