タイトル：<<SERVICE_NAME>>のデザインシステム

あなたはシニアプロダクトデザイナーです。上記サービスのUIを観察した体で、その“らしさ”（色・タイポ・余白・角丸・影・密度・トーン）を抽出し、再現性の高いデザインシステムを作成してください。

# 目的
- 実画面や画面遷移は作らない。デザインシステム（基礎＋部品＋パターン）だけを大量に並べる。
- そのまま実務で流用できる「粒度」「網羅性」「状態」「命名」を最優先にする。

# レイアウト制約（重要）
- “タブ”で切り替える構成にしない。縦に長い「一枚のキャンバス」に、上から下へセクションを積み上げる。
- セクションは見出し＋サンプルの塊で整理し、過剰な囲み枠は使わず、整った余白と揃えで見せる。
- 12カラム相当のグリッドを意識し、すべての要素が一定のリズムで整列するように配置する。

# 生成ボリューム（最低ライン）
- Foundations（基礎）を必ず作る：色 / タイポ / 余白 / 角丸 / 枠線 / 影 / アイコン規約 / レイアウト規約 / モーション規約 / アクセシビリティ規約
- Components（部品）を40種類以上、各部品は「サイズ×バリアント×状態」を揃えて提示する
- Patterns（組み合わせ）を10種類以上（フォーム、検索、空状態、エラー、読み込み、権限不足、テーブル、フィルタ、設定、オンボーディング等）

# 作るべき内容（上からこの順で並べる）
1) Overview
- デザイン原則（3〜5個）＋トーン（硬い/柔らかい等）＋密度（情報量の方針）
- 使ってはいけない例（2〜3個）も小さく提示

2) Foundations
A. Color
- ブランドカラー（濃淡スケール 8〜10段）
- ニュートラル（背景/境界/文字の階調 10段以上）
- ステータス（Success/Warning/Error/Info 各4段以上）
- コントラスト指針（本文・小文字・ボタンの最小コントラスト目安）
- 「用途ベース」の色トークン表（Text/Background/Border/Interactive など）

B. Typography
- フォント方針（和文/英文）＋行間/字間のルール
- スタイルセット（Display, H1-H4, Body L/M/S, Caption, Overline など8〜12種）
- 数字・表・ボタンラベル用の特別ルールも用意

C. Spacing & Layout
- スペーシングスケール（例：2/4/8/12/16/24/32/48…）
- コンポーネント内余白ルール（タップ領域最小、行高、左右パディング）
- 12カラム基準のマージン/ガター例、主要ブレークポイント指針

D. Radius / Border / Shadow
- 角丸スケール（6段以上）＋用途（カード/入力/チップ等）
- ボーダー太さ（1/2…）＋境界色の使い分け
- 影スケール（6段以上）＋Elevationの使い分け（浮き/モーダル/ドロップダウン等）

E. Iconography & Illustration（必要なら）
- 線幅、角、サイズ（16/20/24/32）と配置ルール
- 禁止事項（混在、塗りつぶしの扱い等）

F. Motion（ルールだけでOK）
- 基本のイージング、時間（Fast/Medium/Slow）と用途（ホバー/モーダル/トースト）

3) Components（各コンポーネントは必ず「バリアント×サイズ×状態」を横並びで見せる）
- Button（Primary/Secondary/Tertiary/Ghost/Danger、S/M/L、Default/Hover/Pressed/Disabled/Loading/Focus）
- Icon Button
- Link / Text Button
- Input（Text/Password/Search）＋Prefix/Suffix/Helper/Error
- Textarea
- Select / Combobox
- Checkbox / Radio / Toggle / Segmented
- Slider
- Date input（簡易でOK）
- Chip / Tag / Badge
- Avatar
- Tooltip
- Toast / Snackbar（Success/Warning/Error/Info）
- Modal
- Drawer / Sheet
- Popover
- Dropdown menu
- Pagination
- Breadcrumb
- Progress（Linear/Circular）
- Skeleton
- Card（密度違い3種）
- List item（左アイコン/右アクション/2行）
- Table（ヘッダ/ソート/選択/空状態）
- Search bar（サジェストあり）
- Filter（チップ＋条件パネル）
- Banner / Alert
- Stepper（簡易）
- File upload
- Empty state（数パターン）
- Error state（数パターン）
- Loading state（数パターン）
※他にも不足があれば追加してよい（合計40種以上に到達すること）

各部品に必ず付ける情報（小さくでOK）：
- 使いどころ（Do）
- やってはいけない（Don’t）
- 最小サイズ/推奨文字数/アイコン有無のルール
- キーボードフォーカスの見え方（Focusリング等）

4) Patterns（“部品の寄せ集め”ではなく、実務で使う塊）
- 認証フォーム（ログイン）
- 検索＋結果（0件/読み込み/エラー）
- テーブル＋フィルタ＋バルク操作
- 設定画面のセクション構造（見出し＋説明＋操作）
- オンボーディング（3ステップ例）
- 権限不足/課金誘導の状態
- 通知センター（一覧＋未読）
- プロフィール編集（入力＋バリデーション）
- 破壊的操作の確認フロー
- 空状態カタログ（3種：新規/フィルタ結果なし/エラー）

# クオリティ基準
- すべてのテキストは日本語で、文言も実運用を想定して自然に書く。
- 余白・整列・行間を最優先。見栄えより「使える規約」と「状態網羅」を重視。
- 可能ならライト/ダークの両方を同じ規約で提示（最低でも主要トークンと主要部品は両対応）。

この指示に従い、縦長一枚のキャンバスに、上から順に大量の高品質なデザインシステムを構築してください。
