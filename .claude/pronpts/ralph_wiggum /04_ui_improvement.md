# UI全体改善

## 概要
UI UX Pro Max Skillを使って、既存のUIをレビューし改善します。

## プロンプト

```
/ralph-loop "ポートフォリオサイト全体のUIを改善して。

## タスク
UI UX Pro Max Skillを使って、既存のUIをレビューし改善する。

## プロセス
1. 現在のコンポーネントをスキャン
2. UI UX Pro Maxで改善点を検索
3. 問題点をリストアップ
4. 優先度順に修正
5. 各修正後にビルド確認

## 改善チェックリスト
- [ ] カラーコントラスト（WCAG AA準拠）
- [ ] フォントサイズと行間
- [ ] ボタンのサイズとタップ領域
- [ ] 余白の一貫性
- [ ] アニメーションの適切さ
- [ ] ダークモードの見やすさ
- [ ] モバイルでの使いやすさ

## 禁止事項
- 絵文字をアイコンとして使わない（SVGを使用）
- インラインスタイルを避ける
- !importantを使わない

## 成功基準
- [ ] Lighthouseアクセシビリティ90以上
- [ ] 全ページでレイアウト崩れがない
- [ ] コンソールエラーがない

完了したら <promise>UI_IMPROVED</promise> と出力。" --max-iterations 30 --completion-promise "UI_IMPROVED"
```

## 期待される成果物
- 各コンポーネントの改善
- アクセシビリティの向上
- 一貫したデザインの適用
