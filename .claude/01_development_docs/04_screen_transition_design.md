# 画面遷移設計

## 1. 画面一覧

| 画面名 | パス | 説明 |
|--------|------|------|
| ホーム | `/` | 全セクション統合ページ |
| 作品詳細 | `/works/[id]` | 個別プロジェクト詳細 |
| 404 | `/not-found` | ページ未発見 |
| エラー | `/error` | エラーページ |

---

## 2. 遷移パターン

### 2.1 ホーム → 作品詳細

```
/ (ホーム)
  → Worksセクションの作品カードクリック
    → /works/work-1 (作品詳細)
      → ブラウザバック or ヘッダーロゴクリック
        → / (ホーム)
```

**実装**:
```tsx
import Link from 'next/link';

<Link href={`/works/${work.id}`} className="...">
  <WorkCard work={work} />
</Link>
```

---

## 3. まとめ

画面遷移はシンプルに保ち、ユーザーが迷わないようにする。
